    Chapitre 1: Introduction

    Ce document est un résumé d'un cours d'introduction aux Techniques de l'Information et de la Communication (TIC) .Les TIC regroupent les outils matériels et logiciels qui permettent de manipuler, traiter, produire, faire circuler, transmettre, modifier et échanger des informations. Les TIC ont émergé grâce à la convergence de l'informatique, des télécommunications, de l'électronique et de l'audiovisuel. Les objectifs des TIC sont la connectivité, la modularité, une implantation simple, une utilisation facile et la fiabilité. Les composants des TIC comprennent des matériels tels que les ordinateurs portables, les téléphones portables, les tablettes interactives, les appareils photo numériques, les stations de travail, les serveurs, ainsi que des logiciels tels que les systèmes d'exploitation, les logiciels d'application et les réseaux Internet ou mobiles. Les caractéristiques des TIC incluent la rapidité, la décentralisation du pouvoir, un meilleur accès aux informations, la rationalité et la mondialisation.


    Chapitre 2: Les réseaux informatiques

    Ce document présente les notions de base des réseaux informatiques. Un réseau est un ensemble d'ordinateurs et de périphériques connectés entre eux pour échanger des informations numériques selon des règles bien définies. Les réseaux peuvent être de différentes envergures : LAN, MAN et WAN. Les caractéristiques de base d'un réseau comprennent la topologie, le débit, la distance maximale et le nombre de nœuds maximum pouvant être interconnectés. Les éléments d'un réseau incluent les ordinateurs, les logiciels, les supports et les équipements d'interconnexion. Le document explique également quelques terminologies en réseau telles que la station de travail, le nœud, le serveur et le paquet.


    Chapitre 3: Le modèle OSI

    Le document parle des composants matériels d'un réseau informatique, tels que les cartes réseau, les switches et les hubs. Il explique que les cartes réseau permettent aux ordinateurs de se connecter à un réseau et de communiquer via des protocoles spécifiques tels que TCP/IP. Les switches sont utilisés pour connecter plusieurs appareils sur un réseau local et permettent une communication plus rapide et plus sécurisée, tandis que les hubs ont été remplacés par des switches, car ils ne possèdent pas la fonctionnalité de transmission de données uniquement à l'appareil destinataire. Le document introduit le modèle OSI, qui est un modèle de référence pour les communications de réseau qui divise les communications en sept couches distinctes. Chaque couche est responsable de fonctions spécifiques. Les sept couches sont : la couche physique, la couche liaison de données, la couche réseau, la couche transport, la couche session, la couche présentation et la couche application.


    Chapitre 4: L'Internet et le Web

    Le document est un chapitre sur Internet et le Web. Il explique que le Web est une application d'Internet et permet de consulter des pages accessibles sur des sites, reliées entre elles par des hyperliens. Les aspects techniques du Web incluent les URL, les protocoles HTTP et HTML. Une page Web est un document multimédia stocké dans un fichier HTML et interprété par un navigateur. Un site Web est un regroupement de pages Web autour d'une thématique commune, reliées entre elles par des liens hypertextes et émanant d'une même entité. Enfin, la création d'une page Web HTML nécessite des balises et des principes de langage HTML tels que les titres et les paragraphes.


    Chapitre 5: Système de Gestion de Version

    Ce document aborde le Git, un logiciel de gestion. Les logiciels de gestion de version permettent de conserver un historique des modifications effectuées sur un projet pour faciliter la gestion de projets et travailler en équipe. Git est un leader incontesté dans ce domaine. Le texte explique également les avantages de ce modèle. Git considère les données à la manière d'un flux d'instantanés ou "snapshots", c'est-à-dire qu'à chaque fois qu'un état d'un projet est enregistré, il prend un instantané du contenu de l'espace de travail et l'enregistre localement dans une base de données. Cela permet de réaliser la plupart des opérations localement, sans avoir à être connecté à un serveur central distant, rendant les opérations plus rapides. Un fichier peut avoir deux états dans Git : suivi ou non suivi. Un fichier suivi appartient au dernier instantané capturé par Git et est enregistré dans la base, tandis qu'un fichier non suivi ne l'est pas. Les fichiers sont au départ tous non suivis. Chaque fichier suivi peut avoir trois états : modifié, indexé ou validé. Un fichier modifié est considéré comme modifié tant qu'il n'a pas été indexé. Un fichier est indexé lorsqu'on indique à Git qu'il doit faire partie du prochain instantané dans sa version actuelle. Le document explique l'utilisation de Git pour la gestion de versions dans les projets de développement de logiciels. Il décrit d'abord les concepts de base de Git, notamment les états de fichiers, les zones de travail et le répertoire Git, avant de donner des instructions détaillées pour la création d'un dépôt Git à partir d'un projet existant et la gestion de versions de fichiers à l'aide de Git.
